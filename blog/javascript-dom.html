JavaScript and the DOM

If CSS is the peanut butter and jelly, and HTML is the bread, JavaScript is your knife and fork. Another great analogy, because you would totally use a knife and fork to cut a pb&j sandwich. But anyway, right now we're in an analogy with a weird person who cuts their sandwiches with a knife and fork.

JS (JavaScript) gives the code MOTION. HTML contains a bunch of static components, JS can turn them into static elements with areas that have MOTION. That are alive and beating. CSS can only make stuff look pretty, JavaScript makes a website or app compelling and worth talking about.

Something that's hard af about JavaScript is Loops. Loops loops loops. My nemesis. A loop is like drinking a nice, hot coffee. Except way less pleasant. Basically if I were to drink a coffee, I'd drink it in the same way again and again until the coffee is finished. That's basically what looping does. The i or Iterator counts how many times it has gone through the loop, and it keeps counting until it meets the constraints you give it. Most loops end in the same way with i++, which basically just means the loop keeps going until it's told to stop. Once the rules are set for the iterator we give it a function, the function in our analogy would be to drink the coffee. Within the "for" brackets we would set a rule that tells the Iterator (you, I, whoever) to keep sipping until there's nothing left. At the same time, for some reason, I'm also counting how many times I've sipped because normally it takes me 20 sips to finish a coffee. Or something like that.

Here is a loop if we used the analogy. for (let YOU DRINK = I'M COUNTING; YOU DRINK = THE SIZE OF THE COFFEE; i++), then below that code there would be an area which i do not completely understand yet. But if we're using the coffee analogy it would do something like tell the function to search the your brain (or the HTML) for signals that you want to drink a coffee.

The DOM is what happens when you press F12 on the keyboard. In the SIMPLEST way possible, that area that appears in the browser is the DOM. You can use it to test stuff in the browser, see what your JS code is doing, search for elements to target with your JS code, and it will even tell you what errors can be found in your code.
